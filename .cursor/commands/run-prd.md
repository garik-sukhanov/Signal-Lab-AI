# /run-prd

Запусти реализацию PRD через orchestrator-подход.

## Вход
- Путь к PRD файлу (например `docs/prds/003_prd-cursor-ai-layer.md`).
- Ограничения по сроку/приоритетам (если есть).

## Шаги
1. Прочитай PRD и выдели требования/критерии приёмки.
2. Проведи быстрый scan codebase и gap analysis.
3. Сформируй поэтапный plan с atomic задачами.
4. Выполни задачи партиями: сначала low-risk, затем integration точки.
5. После каждого этапа делай проверку соответствия acceptance criteria.
6. В конце дай отчёт: что сделано, что осталось, риски.

## Требования к качеству
- Делай минимальные изменения с максимальной предсказуемостью.
- Следуй `.cursor/rules/*` и используй доступные skills.
- Указывай next steps только если критерии не закрыты полностью.
